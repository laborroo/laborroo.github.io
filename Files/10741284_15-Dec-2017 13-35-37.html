Verify Salary  [UC-158 Use Case]
     Open in Win Client    
Description (Goal in Context) This UC documents the process of verifying a Member's Salary throgh both his Payslip and Bank Statement from the Last three months. The salary and on the Payslip must reflect in the provided bank statement. It is this same bank account that must be used to setup payment
Primary Actors TopTeam
 ACTR-152 Member
Supporting Actors TopTeam
 ACTR-143 Admin
Pre Conditions
Must be logged in
Must be an Unverified Member
Post Conditions
Success end condition
- Member 's Salary is verified
- If Employment has already been verified then the Member becomes an Verified Member  
Failure end condition
- Member 's Salary is not verified and still remain an Unverified Member
Minimal Guarantee
-

Flows
Main Flow-of-Events
  1. Member requests to verify Salary
  2. System asks Member to upload Payslip and Bank Statement
  3. Member uploads Payslip and Bank Statement
  4. System notified Admin that there is a new Salary verification request
  5. Admin checks the documents manually
  6. Admin updates Member 's account to show that Salary has been verified.
  7. Use Case ends with Success.

Alternate Flows 
5a. Verification fails by Admin
  1. Admin fails to verify documents specify the reason for the failure.
  2. System notifies Member that the verification failed and sends the Member a reason for the failure
  3. Use Case ends with Failure.
6a. User already has employment verified
  1. System changes account status to Verified Member
  2. Use Case ends with Success.
