Register Account  [UC-151 Use Case]
     Open in Win Client    
Description (Goal in Context) This UC describes the process of a guest registering for an account on the Marketplace. An email address, and password is required to create an account. An email with an activation link will be sent to the provided email.
Primary Actors TopTeam
 ACTR-150 Guest
Supporting Actors TopTeam
Post Conditions
Success end condition
- An account will be created and an Email sent to the provided email address

Failure end condition
- Email already exists or Registration is cancelled. No new account created

Minimal Guarantee
-

Flows
Main Flow-of-Events
  1. Guest choose to register new Account
  2. System displays a Registration Page
  3. Guest enters Email Address and Password
  4. System verifies Email Address doesn't exist and Password meets criteria
  5. System creates new account and send activation email to provided email address
  6. Use Case ends with Success.

Extensions 
4a. Email Address already exists
  1. System detects Email Address already exists
  2. System notifies Guest that Email Address already exists
  3. Continue from step 3. of Main Flow-of-Events.
